Workflow
  1. Create or load a DDL definition in the top-left panel
  2. Select a Volume / Subvolume / DDL in the tree to set the parse scope
  3. Paste or drop a message in the top-right panel
  4. Click ▶ Parse — results appear below
DDL format
# FIELD_ID   TYPE   LENGTH   "Description"
MTI    FIXED   4   "Message Type Indicator"
BMP    FIXED   8   "Primary Bitmap"
2      LLVAR   19   "Primary Account Number"
48    LLLVAR 999 "Additional Data"
Types: FIXED LLVAR LLLVAR BMP
Message formats accepted
  • Pure hex — F0200022000000...
  • Spaced hex — F0 20 00 22 ...
  • Tandem dump — 0000: F0 F2 .. |0200|
  • ASCII raw — character bytes as-is
Separate multiple messages with a blank line, or use ISO timestamp prefixes — they are auto-detected.
DDL tree — right-click actions
  • Right-click empty area → New Volume / Import… / Export All…
  • Right-click Volume → New Subvolume / Copy / Export Volume… / Rename / Delete
  • Right-click Subvolume → New DDL / Copy / Export Subvolume… / Rename / Delete
  • Right-click DDL → Copy / Export DDL… / Rename / Delete
Export / Import
  • Export saves a .json file — choose exactly which items to include via the checklist
  • Import reads a previously exported .json file — conflicts are highlighted and will be overwritten on confirm
Drag & Drop
  • Drop any text file onto the Message Input area
  • Drop a DDL file onto the DDL Editor to preview, then Save
  • Drop a DDL file onto the tree pane to import directly
Parse results
  • Click a row → highlights the field bytes in the hex dump
  • Check Trk on fields you want to compare across messages
  • Click ⊞ Track Mode to see a side-by-side cross-message matrix
  • Use < > to navigate through multiple messages
